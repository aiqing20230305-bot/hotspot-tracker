## 一、🔴 index.html UI/UX 问题

### P0 - 严重问题

#### 1. 🐛 平台筛选器完全失效
- **现状：** 数据中平台字段为组合格式（如 `微博/抖音`、`小红书/抖音`），共30条热点中无一条匹配 `抖音`、`微博`、`小红书` 单独的筛选条件
- **影响：** 平台筛选按钮点击后返回空列表，核心功能不可用
- **修复方案：**
  ```javascript
  // 修改 filterByPlatform 函数
  function filterByPlatform(platform) {
      if (platform === 'all') { currentFilter = 'all'; }
      else {
          currentFilter = platform;
          // 改为 includes 模糊匹配
          filteredTopics = allTopics.filter(t => 
              (t.platform || '').includes(platform)
          );
      }
  }
  ```

#### 2. 🐛 热点逻辑分析（Logic）永远不显示
- **现状：** 30条热点中 `logic` 字段全部缺失（0/30），UI 中 `topic.logic` 判断永远为 false
- **影响：** "热点背后逻辑" 区域形同虚设，用户看不到任何洞察
- **修复方案：** 后端 Python 在生成 JSON 时必须填充 `logic` 字段

#### 3. 🐛 趋势标记全部缺失
- **现状：** 30条热点中 `trend` 字段全部缺失（0/30），前端硬编码为 `🔥 稳定`
- **影响：** 快捷筛选"热度上升"永远显示0，趋势分析无意义
- **修复方案：** 后端需要计算并填充趋势标记

#### 4. 🐛 XSS 注入风险
- **现状：** 热点标题直接嵌入 HTML 和 onclick 属性中，未经转义：
  ```javascript
  onclick="toggleFavorite('${topic.title}', '${topic.title.replace(/'/g, "\\'")}')"
  ```
- **影响：** 恶意标题可能执行任意 JS
- **修复方案：** 使用 `data-` 属性 + `textContent` 渲染，或引入 DOMPurify

### P1 - 重要问题

#### 5. 📦 文件过于臃肿（5450行 / ~180KB）
- **问题：** 单文件包含 ~3000行 CSS + ~2000行 JS + ~450行 HTML
- **影响：** 首次加载慢、无法缓存、维护困难
- **建议：** 拆分为 `index.css`、`app.js`、`charts.js`、`index.html`

#### 6. 📊 三套重复图表区域并存

- **影响：** 页面信息过载，三个区域展示类似数据
- **建议：** 合并为"数据概览"和"深度分析"两个层级

#### 7. 🎨 相关选题推荐使用硬编码 Mock 数据
- **现状：** `RELATED_TOPICS_MOCK` 是写死的5条数据，所有热点显示相同的推荐
- **修复方案：** 关联 `client_ideas.json` 中的真实选题数据

#### 8. 📈 Mini 图表使用伪随机数据
- **现状：** `generateChartData()` 基于标题字符的 hash 生成数据，完全虚假
- **修复方案：** 接入真实的内容分析数据
